Introduction

EDUDONOR is a comprehensive web-based platform designed to bridge the gap between blood donors and recipients. It facilitates seamless management of blood donation camps, donor registration, and emergency blood requests.

Goal: To save lives by streamlining the blood donation process.

Relevance of Project

  • Addresses critical blood shortages efficiently.
  • Reduces response time in emergencies.
  • Digitalizes manual record-keeping for camps and donors.
  • Promotes awareness and accessibility.

Methodology

Developing a life-saving digital ecosystem step by step.

Phase 1: Planning

  • Requirement Gathering: Identifying the needs of donors, recipients, and admins.
  • Scope Definition: Establishing project boundaries and feature sets.
  • Timeline Estimation: Planning the development lifecycle.

Phase 2: Analysis

  • System Architecture: Designing a robust MERN stack infrastructure.
  • Database Schema: Structuring MongoDB collections for speed and security.
  • API Design: Creating clean, efficient endpoints for data flow.

Phase 3: Testing

  • Unit Testing: Verifying server logic using Jest.
  • Linting: Maintaining code quality with ESLint/Prettier.
  • UI Validation: Manual and automated interface verification.

Phase 4: Implementation

  • Backend: Developing Node.js/Express server and REST APIs.
  • Frontend: Building a responsive React dashboard.
  • Security: Implementing JWT authentication and data protection.

Platform Roles

Empowering every stakeholder in the donation lifecycle.

Donor Role

  • Registration: Seamless sign-up and professional profile management.
  • Eligibility: Automated health checks and donation interval tracking.
  • Camp Participation: Find and register for local donation camps.
  • Urgent Alerts: Receive instant notifications for critical blood needs.

Recipient Role

  • Blood Requests: Direct posting of urgent requirements with hospital data.
  • Smart Matching: Instant lookup of local donors by blood group availability.
  • Live Tracking: Real-time status updates on donor acceptances.
  • History: Comprehensive log of received donations and request status.

Admin Role

  • Camp Management: Full control over scheduling and venue logistics.
  • User Verification: Ensuring platform security through donor/recipient vetting.
  • Analytics: High-level oversight of donation trends and successful matches.
  • System Controls: Managing platform states, notifications, and core alerts.

DFD: Operational View

  • Donors: Register and providing eligibility data.
  • Admin: Co-ordinating camps and inventory management.
  • Recipients: Requesting blood and track statuses.
  • System: Backend logic secure data matching.

Findings & Recommendation

Findings: The digital platform significantly reduces time to find donors compared to traditional methods.

Recommendation:

  • Implement real-time notifications (SMS/Email) for urgent requests.
  • Expand to mobile application for better accessibility.
  • Integrate location-based services for finding nearest camps.

Conclusion

EDUDONOR successfully demonstrates how technology can be leveraged for social good. By organizing and simplifying the blood donation process, it creates a reliable network that can save countless lives.
